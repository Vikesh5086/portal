## DATABASE SCHEMA (SQLite)

```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY,
  college_id TEXT UNIQUE NOT NULL,
  password TEXT NOT NULL,
  name TEXT NOT NULL,
  role TEXT DEFAULT 'student'  -- 'student' or 'admin'
);

CREATE TABLE courses (
  id INTEGER PRIMARY KEY,
  course_title TEXT NOT NULL,
  course_id TEXT NOT NULL,
  class_nbr TEXT,
  subject_catalog_nbr TEXT,
  class_section TEXT DEFAULT '1R1'
);

CREATE TABLE student_courses (
  id INTEGER PRIMARY KEY,
  student_college_id TEXT NOT NULL,
  course_id INTEGER NOT NULL,
  FOREIGN KEY (course_id) REFERENCES courses(id)
);

CREATE TABLE assignments (
  id INTEGER PRIMARY KEY,
  course_id INTEGER NOT NULL,
  assignment_name TEXT NOT NULL,
  category TEXT NOT NULL,  -- MST, Quiz 1, Quiz 2, TUT1, EST, etc.
  begin_date TEXT,
  due_date TEXT,
  max_marks REAL NOT NULL,
  FOREIGN KEY (course_id) REFERENCES courses(id)
);

CREATE TABLE marks (
  id INTEGER PRIMARY KEY,
  student_college_id TEXT NOT NULL,
  assignment_id INTEGER NOT NULL,
  marks_obtained REAL,
  FOREIGN KEY (assignment_id) REFERENCES assignments(id)
);
```

## ADMIN PANEL (admin.html) — accessible only to admin role users

Admin login uses same login page but role = 'admin' in DB.

Admin dashboard shows:
- List of all students (name + college ID)
- Select a student → shows their enrolled courses
- Select a course → shows all assignments for that course
- For each assignment: input field to enter/update marks for that student
- Save button → updates marks in DB

Admin can also:
- Add new students
- Add new courses
- Enroll a student in a course
- Add assignments to a course (name, category, max_marks, dates)

---

## GRADE CALCULATION LOGIC

For each course, calculate:
- Current Mid-Term Grade = (MST marks / MST max) * 100
- Current Overall Grade = weighted sum:
  - MST: 30%
  - Quiz 1: 11% of total (scaled)
  - Quiz 2: 14% of total (scaled)  
  - TUT1: 5% of total (scaled)
  - EST: 40%
  (adjust weights based on actual max_marks in DB)
- Display as percentage with 2 decimal places

## API ROUTES

POST /api/login — authenticate user, set session
GET /api/logout — clear session
GET /api/student/courses — get logged-in student's courses
GET /api/student/course/:courseId/grades — get assignments + marks for a course
POST /api/admin/marks — admin updates marks { student_college_id, assignment_id, marks }
POST /api/admin/student — admin adds student
POST /api/admin/course — admin adds course
POST /api/admin/enroll — admin enrolls student in course
POST /api/admin/assignment — admin adds assignment to course
GET /api/admin/students — list all students
GET /api/admin/student/:id/courses — get student's courses
